    128. 最长连续序列

    给定一个未排序的整数数组 nums ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

    请你设计并实现时间复杂度为 O(n) 的算法解决此问题。

    示例 1：

    输入：nums = [100,4,200,1,3,2]
输出：4
解释：最长数字连续序列是 [1, 2, 3, 4]。它的长度为 4。

    示例 2：

    输入：nums = [0,3,7,2,5,8,4,6,0,1]
输出：9

    提示：

    • 0 <= nums.length <= 105

    • -109 <= nums[i] <= 109

    为了在 $O(n)$ 的时间复杂度内找出未排序整数数组 nums 中数字连续的最长序列的长度，我们可以使用哈希表来优化查找过程。基本思路是先将数组中的所有元素存入哈希表中，方便后续快速判断某个元素是否存在。然后遍历数组，对于每个元素，如果它是一个连续序列的起始元素（即该元素减 1 不在哈希表中），就从这个元素开始不断尝试寻找连续的下一个元素，统计连续序列的长度，并更新最长连续序列的长度。

    以下是使用 Python 实现的代码：

    def longestConsecutive(nums):
    # 将数组元素存入集合中，方便快速查找
    num_set = set(nums)
    longest_streak = 0

    for num in num_set:
        # 检查当前元素是否是连续序列的起始元素
        if num - 1 not in num_set:
            current_num = num
            current_streak = 1

            # 不断尝试寻找连续的下一个元素
            while current_num + 1 in num_set:
                current_num += 1
                current_streak += 1

            # 更新最长连续序列的长度
            longest_streak = max(longest_streak, current_streak)

    return longest_streak

# 测试示例
nums1 = [100, 4, 200, 1, 3, 2]
print(longestConsecutive(nums1))  # 输出: 4

nums2 = [0, 3, 7, 2, 5, 8, 4, 6, 0, 1]
print(longestConsecutive(nums2))  # 输出: 9

    复杂度分析

    • 时间复杂度：$O(n)$。虽然代码中有嵌套循环，但每个元素最多被访问两次。外层循环遍历集合中的每个元素，而内层循环在寻找连续序列时，对于每个元素只会在它是连续序列的起始元素时才会执行，因此总体时间复杂度是线性的。
    • 空间复杂度：$O(n)$。主要用于存储集合，集合中存储了数组中的所有元素。

    代码解释

    1. 创建集合：使用 set(nums) 将数组中的元素存入集合 num_set 中，这样可以在 $O(1)$ 的时间复杂度内判断某个元素是否存在。
    2. 遍历集合：对于集合中的每个元素 num，检查 num - 1 是否不在集合中，如果是，则说明 num 是一个连续序列的起始元素。
    3. 寻找连续序列：从起始元素 num 开始，不断尝试寻找连续的下一个元素，即 current_num + 1，并更新当前连续序列的长度 current_streak。
    4. 更新最长连续序列长度：在每次找到一个连续序列后，使用 max(longest_streak, current_streak) 更新最长连续序列的长度。
    5. 返回结果：最终返回最长连续序列的长度。

# 测试示例
test_cases = [
    # 示例 1
    ([100, 4, 200, 1, 3, 2], 4),
    # 示例 2
    ([0, 3, 7, 2, 5, 8, 4, 6, 0, 1], 9),
    # 空数组
    ([], 0),
    # 只有一个元素的数组
    ([5], 1),
    # 所有元素都相同的数组
    ([2, 2, 2, 2], 1),
    # 连续元素的数组
    ([1, 2, 3, 4, 5], 5),
    # 不连续且无连续序列的数组
    ([1, 3, 5, 7, 9], 1),
    # 包含负数的数组
    ([-3, -2, -1, 0, 1], 5),
    # 多个连续序列的数组
    ([1, 2, 3, 5, 6, 7, 9, 10, 11], 3),
    # 序列中有间隔的数组
    ([1, 2, 4, 5, 7, 8], 2)
]

for nums, expected in test_cases:
    result = longestConsecutive(nums)
    if result == expected:
        print(f"测试通过: 输入 {nums}, 预期输出 {expected}, 实际输出 {result}")
    else:
        print(f"测试失败: 输入 {nums}, 预期输出 {expected}, 实际输出 {result}")

    代码解释

    1. 定义测试用例：test_cases 列表中存储了多个元组，每个元组包含一个测试输入数组和对应的预期输出结果。这些测试用例涵盖了空数组、单元素数组、元素全相同数组、连续元素数组、不连续无连续序列数组、包含负数数组、多个连续序列数组以及序列中有间隔的数组等多种情况。
    2. 执行测试：通过遍历 test_cases 列表，对每个测试用例调用 longestConsecutive 函数得到实际输出结果，并与预期输出进行比较。
    3. 输出测试结果：根据比较结果输出测试通过或失败的信息，方便你查看函数在不同输入下的表现。
